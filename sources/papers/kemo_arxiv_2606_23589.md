# KEMO: Event-Driven Keyframe Memory for Long-Horizon Robot Manipulation with VLA Policies

> 来源归档（ingest）

- **标题：** KEMO: Event-Driven Keyframe Memory for Long-Horizon Robot Manipulation with VLA Policies
- **类型：** paper
- **来源：** arXiv abs / HTML；项目页交叉核对
- **原始链接：**
  - <https://arxiv.org/abs/2606.23589>
  - <https://arxiv.org/html/2606.23589v1>
  - <https://hatty-z.github.io/KEMO>
- **作者：** Yihan Zeng, Minghao Ye, Yiyuan Chen, Yide Shentu, Philipp Wu, Zike Yan, Zhongyu Li
- **机构：** 香港具身智能实验室（Hong Kong Embodied AI Lab）、香港中文大学（CUHK）、xdof.ai、电子科技大学（UESTC）、上海交通大学（SJTU）
- **入库日期：** 2026-07-08
- **一句话说明：** 面向长程记忆依赖双臂操作，提出可插拔 **事件驱动关键帧记忆**：运动学显著性 + DINOv2 视觉去重选帧 → SigLIP 编码入 memory bank → 与当前视觉 **masked cross-attention + gated residual fusion** 注入 **π₀.₅**；关键帧邻域 **loss 加权** 与记忆内容同源；真机 6 任务（28–95 s、830–2846 步）聚合 TSR **+23.6 pt**、SCR **+34.1 pt** 相对无记忆 π₀.₅，优于 MemoryVLA。

## 核心论文摘录（MVP）

### 1) 问题：阶段歧义与部分可观测

- **链接：** <https://arxiv.org/abs/2606.23589> §I
- **摘录要点：** 长程操作常出现 **stage ambiguity**：当前视觉相似但应执行不同动作，取决于已完成子步骤。仅条件于当前观测的 VLA 难以在歧义点选对动作。现有记忆增强路线两类局限：**稠密历史**（token merge / RNN / 固定潜表示）易把稀疏状态转移与大量冗余帧混压；**任务分解**（VLM 跟踪进度）需子任务监督、在线推理成本高且语义摘要/子目标幻觉可传播。

### 2) 事件驱动关键帧检测（无任务标注、无在线 VLM）

- **链接：** arXiv §3.3；Appendix D
- **摘录要点：**
  - **事件显著性：** 关节位移滑动均值 $\bar{\delta}_t$ 得 $S_t=1/(1+\bar{\delta}_t)$，减速峰对应抓放、覆盖、释放等过渡。
  - **因果峰检测：** 在线确认局部极大候选关键帧。
  - **视觉去重：** 冻结 **DINOv2** 嵌入，候选与上一接受帧余弦距离 $v_i>\epsilon$（$\epsilon=0.05$）才保留；否则视为无场景变化的运动停顿。
  - **不应期** $r$ 抑制同一次过渡重复触发；$w$、峰窗、$r$ 按任务调参（表 6）。
- **对 wiki 的映射：**

  - [机器人关键帧与运动编辑工具](../../wiki/entities/robot-motion-keyframe-editors.md) — 关键帧抽象对照（离线示教 vs 在线事件检测）。

### 3) 关键帧记忆注入与门控融合

- **链接：** arXiv §3.4
- **摘录要点：**
  - 接受帧用与骨干相同的 **SigLIP** 编码，$4\times4$ 空间池化将每帧 **256→16** token；bank 保留最近 **$K$** 个关键帧（不足则 pad + mask）。
  - **可学习时序位置嵌入** 区分阶段顺序。
  - 当前帧 patch token 为 Query，bank 为 KV；**masked cross-attention** 后 **gated residual**：$\hat{X}_t=X_t+\sigma(g([X_t;X'_t]))\cdot X'_t$，门控负偏置初始化以训练初期保持原 π₀.₅ 行为。
  - 在 **视觉编码阶段** 注入，不改 LLM 前缀长度与架构。
- **对 wiki 的映射：**
  - [KEMO](../../wiki/entities/paper-kemo-event-driven-keyframe-memory-vla.md) — 记忆融合表。
  - [π₀.₇ Policy](../../wiki/methods/pi07-policy.md)、[VLA](../../wiki/methods/vla.md) — π 系骨干与记忆增强对照。

### 4) 关键帧一致训练（loss 加权）

- **链接：** arXiv §3.5
- **摘录要点：** 关键帧索引同时决定 **$\mathcal{M}_t$ 内容** 与 **监督权重** $w_t$：距任一关键帧 $\leq\delta$ 的 timestep 将 flow-matching loss 乘 $\lambda=8$（$\delta=3$），其余为 1。推理记忆与训练强调同源，促使策略在将被存入记忆的事件邻域更准确。

### 5) 真机六任务与定量结果

- **链接：** arXiv §4；项目页
- **摘录要点：**
  - **平台：** **YAM 双臂** leader–follower 遥操作采 100–200 demo；三视角 **Decxin RGB**（顶视 + 双腕）；推理仅 follower 自主执行。
  - **任务：** Swap Foods（2 子任务，28 s）、Find Block（4）、Cover Blocks（6）、Box Refill（4）、Make Sandwich（4）、Drawer Items Replacement（4 计分阶段，95 s）。
  - **指标：** 每任务 **12 trials**；**TSR** 全阶段顺序成功；**SCR** 从首阶段起连续完成阶段数。
  - **聚合：** KEMO TSR **51.4%** vs π₀.₅ **27.8%**（**+23.6 pt**）；SCR **76.4%** vs **42.3%**（**+34.1 pt**）；vs MemoryVLA TSR **+50.0 pt**、SCR **+60.9 pt**。
  - **Cover Blocks 消融：** 事件关键帧 TSR **25%** > 均匀采样 **0%** > 近帧 **8.3%**；去门控 TSR **0%**；去 loss 加权 TSR **25%** → 完整 **75%**（9/12）。

### 6) 局限与相邻记忆 VLA 对照

- **链接：** arXiv §5；§2
- **摘录要点：** $w$、峰窗、$r$、bank 容量 $K$ 需任务级调参；未来方向为可学习事件选择与动态 bank。相对 **MemoryVLA**（per-step token merge）、**ContextVLA/HAMLET/ReMem-VLA**（稠密/压缩历史）、**MemER/BPP**（VLM 关键帧）等，KEMO 强调 **稀疏事件帧 + 轻量运动学检测 + 插拔式视觉融合**，避免在线 VLM 分解。
