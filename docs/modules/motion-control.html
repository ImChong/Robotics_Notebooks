运动控制模块

这是 `Robotics_Notebooks` 的标准模块页模板示例。目标不是只解释概念，而是把“问题、依赖、方法、产出、资源、关联模块”串起来。

这个模块解决什么问题

让机器人在给定任务和约束下，产生稳定、可控、可执行的身体运动行为。

为什么对人形机器人重要

运动控制是人形机器人的硬骨架。没有它，强化学习和模仿学习再花，也只是飘在空中的策略。

前置依赖

  • 线性代数
  • 刚体运动与坐标变换
  • 运动学
  • 动力学
  • 优化基础

核心子主题

  • 状态估计
  • PID / LQR
  • MPC
  • QP-WBC / TSID
  • 人形 locomotion

学完后应该会什么

理解骨架

知道一个人形机器人控制系统从状态估计、模型、约束到控制输出怎么串起来。

读懂论文

能读懂 WBC、MPC、locomotion 和 humanoid control 相关论文里的核心结构。

跑通 baseline

能在仿真中搭起基本控制闭环，并对控制表现做基础分析。

连接学习算法

知道控制主线如何和 RL / IL 结合，而不是把两者割裂看待。

最小落地任务

任务 1

用 Pinocchio 或类似工具跑通一个人形模型的正逆运动学。

任务 2

实现一个简单倒立摆或二阶系统的 LQR / MPC 控制。

任务 3

读一篇 humanoid locomotion / WBC 论文，并把控制结构画成框图。
