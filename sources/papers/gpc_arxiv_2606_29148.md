# GPC: Large-Scale Generative Pretraining for Transferable Motor Control（arXiv:2606.29148）

> 来源归档（ingest）

- **标题：** GPC: Large-Scale Generative Pretraining for Transferable Motor Control（简称 **GPC**，Generative Pretrained Controllers）
- **类型：** paper / physics-based character animation / generative controller / discrete motion token / RL / PEFT
- **arXiv：** <https://arxiv.org/abs/2606.29148>
- **PDF：** <https://arxiv.org/pdf/2606.29148>
- **会议：** SIGGRAPH Conference Papers 2026（Los Angeles，2026-07-19–23）
- **DOI：** <https://doi.org/10.1145/3799902.3811038>
- **作者：** Yi Shi, Yifeng Jiang, Chen Tessler, Xue Bin Peng
- **机构：** 西蒙菲莎大学（SFU）、英伟达（NVIDIA）
- **入库日期：** 2026-07-01
- **一句话说明：** 将运动技能 **离散 token 化**（FSQ + 端到端 RL 跟踪），再用 **GPT 式 Transformer** 做 **下一 token 预测** 训练生成式物理控制器；在 **>600 h** 动作数据上预训练后可在物理仿真中 **实时** 采样自然行为，并通过 **CoLA（Conditional Low-rank Adaptation）** 参数高效微调到转向、轨迹跟踪、路点、障碍与平台跳跃等下游任务。

## 摘要级要点

- **问题：** 连续潜空间生成式控制器（VAE/GAN/CVAE 等）易 **mode collapse、离流形漂移**；VQ-VAE 离散化又常需 **codebook 启发式**，且多在 **~20 h 级** 数据上验证，难扩到 **数百小时** 技能库。
- **核心思路：** 把「GPT 预训练 + 微调」范式搬到 **物理仿真角色控制**——先学 **运动词汇表**（discrete skill tokens），再学 **token 序列分布**，最后 **PEFT 适配新任务**。
- **三阶段流水线：**
  1. **Skill Quantization：** FSQ encoder–decoder + PPO **端到端 RL** 跟踪参考运动；隐式词表规模 $L^d$（实验 $L{=}9, d{=}40$ → $9^{40}$）。
  2. **Generative Controller：** 将每步 $d$ 个 $L$-ary token **分组**（grouping $G$）压缩序列长度；GPT decoder + teacher forcing 交叉熵做 **下一 grouped token 预测**；推理 nucleus (top-$p$) 采样。
  3. **Task Adaptation：** 冻结预训练生成控制器 + **CoLA** 层；**SFT**（示例运动交叉熵偏置技能子集）+ **RLFT**（PPO，动作空间为离散 skill token 序列）。
- **规模与实时性：** **>600 h**（BONES 主训练）；消费级 **RTX 4090** 可实时运行；$G{=}4$ 时约 **115 FPS**（Table 3）。
- **跟踪精度：** Bones 上 FSQ 端到端 RL **99.98%** 成功率（MPJPE 34.90 mm），优于 VQ-VAE（99.94% / 37.92 mm）；相对「先运动学监督训 encoder 再冻结核」的 FSQ-K 优势显著（99.03% / 78.26 mm）。
- **涌现行为：** 无条件采样可产生翻滚、跳跃、舞蹈、杂技等；对外力扰动有 **类人恢复**（侧手翻、前滚起身等），**无需专门恢复奖励**。
- **下游任务：** 摇杆转向、轨迹跟踪、目标到达、障碍/平台跳跃与爬行等；相对 CVAE 任务控制器保留 **随机性与行为多样性**（同任务多次 rollout 轨迹明显不同）。
- **局限（论文 §9）：** 以 **locomotion 类任务** 为主；尚无 **文本条件** 与 **人–物交互**；面向 **物理角色动画** 而非人形机器人真机部署。

## 核心摘录（面向 wiki 编译）

### 1) FSQ 端到端技能量化（§5）

- **机制：** encoder 将未来 $h$ 帧参考状态 $\hat{\mathbf{s}}_{t:t+h}$ 映射为连续 $\mathbf{z}_t$，逐维 FSQ 量化；decoder 以 $(\mathbf{s}_t, \hat{\mathbf{z}}_t)$ 输出 PD 目标关节旋转；全链路 PPO + straight-through estimator。
- **相对 VQ-VAE：** 无需 EMA codebook、dead-code 重初始化等启发式；Table 1 显示同等协议下 **更高成功率、更低 MPJPE**。

### 2) GPT 式生成式控制器（§6）

- **机制：** grouped token 自回归因子分解 $p(\tilde{\mathbf{z}}_t \mid \mathbf{s}_t)$；因果 Transformer + 交叉熵；推理每步 nucleus 采样后送冻结 FSQ decoder 出动作。
- **设计权衡：** $G{=}5$ 多样性（APD）最高但 ADE/加速度略差；$G{=}4$ 在质量与 **115 FPS** 间平衡（Table 3）。

### 3) CoLA 任务适配：SFT + RLFT（§7–8）

- **CoLA：** 在 DoRA 式幅度–方向分解的低秩分支上叠加 **FiLM 任务条件** $\gamma(\mathbf{c}), \beta(\mathbf{c})$，新增参数 **<1%**。
- **SFT：** 用示例运动交叉熵 **收窄** 技能子集（如蹲行风格）；降低策略熵但不损任务回报与扰动鲁棒性（Table 4）。
- **RLFT：** PPO 优化任务奖励；探索用 **无条件生成先验 + nucleus 采样** 正则，保留自然行为。

### 4) 实验设置与局限（§8–9）

- **基础设施：** Isaac Gym + ProtoMotions；BONES 训练 24×A100，其余单卡 A100；部署 RTX 4090 实时。
- **与机器人栈关系：** 同属 **Xue Bin Peng / NVIDIA** 物理角色控制谱系（DeepMimic → ASE → ProtoMotions）；GPC 把 **离散 token + Transformer 预训练** 推到 **600 h+** 规模，但论文明确 **动画角色** 语境，与 [SONIC](../../wiki/methods/sonic-motion-tracking.md) / [BFM](../../wiki/entities/paper-behavior-foundation-model-humanoid.md) 的人形真机 WBC 是 **互补对照**（仿真动画 vs 机器人部署）。
