# GaP: A Graph-as-Policy Multi-Agent Self-Learning Harness For Variational Automation (VA) Tasks（arXiv:2607.05369）

> 来源归档（ingest）

- **标题：** GaP: A Graph-as-Policy Multi-Agent Self-Learning Harness For Variational Automation (VA) Tasks
- **类型：** paper / agentic-robotics / code-as-policy / variational-automation / manipulation / industrial-automation
- **arXiv abs：** <https://arxiv.org/abs/2607.05369>
- **arXiv HTML：** <https://arxiv.org/html/2607.05369v1>
- **PDF：** <https://arxiv.org/pdf/2607.05369>
- **项目页：** <https://graph-robots.github.io/gap/>
- **代码：** <https://github.com/graph-robots/graph-as-policy>（Beta，2026-07-01；技能库 <https://github.com/graph-robots/open-robot-skills>）
- **机构：** 加州大学伯克利分校（University of California, Berkeley）、NVIDIA、卡内基梅隆大学（Carnegie Mellon University）、博世（Bosch）
- **入库日期：** 2026-07-08
- **一句话说明：** **Graph-as-Policy（GaP）** 面向 **变体自动化（VA）** 任务：多 agent 编码 harness 从 **MORSL** 技能库合成 **有向计算图**（类 ROS/TAMP），在 **NVIDIA Isaac** 内仿真排练中自学习优化图结构与参数，再以 edge 解释器持久执行；8 项开放 VA benchmark 上显著优于 VLA、TAMP 与单 agent CaP 基线，且可将 VLA **staging** 进分布后成功率 **>2×**。

## 摘要级要点

- **问题：** 商业/工业场景需要 **持久、可靠** 的机器人自动化；**固定自动化（FA）** 可靠但无几何/位姿适应；**通才机器人（GR）** 的 model-free **VLA** 泛化强但 **尚不可靠**；中间带 **对象几何与位姿有界变化** 的 **VA** 任务缺乏系统化学习范式。
- **核心范式：** 策略 = **有向计算图** $\mathcal{G}=(V,E)$：原子节点（感知/规划/控制/脚本路由）+ 数据边与控制边；受 **ROS 计算图** 与 **TAMP 层次结构** 启发，兼顾 **可解释、模块化、可复用**。
- **多 agent harness：** **Orchestration Agent** 按语义切段 → **Skill Agents** 为各段合成局部子图 → 静态 **类型检查** 后拼成可执行图；**生成与测试分离** 限制上下文、抑制 agent「作弊」。
- **MORSL：** **Modular Open Robot Skill Library**，首发 **51** 项技能（感知、抓取规划、运动规划、2D/3D 视觉、ROS 翻译、验证等），声明图 IO、参数与前置条件。
- **Self-learning rehearsal：** 参数化仿真并行排练多 task instance；注册节点前后状态 + 接触反馈 **定位失败节点**；agent 编辑拓扑/参数直至成功率与吞吐 plateau；Make Popcorn **33% → 94%（sim）/ 90%（real，18/20）**。
- **8 项 VA benchmark：** 仿真 4 + 真机 4——杂货履约/打包、做爆米花、USB-C 插线（UR5+力反馈）、工业洗箱（双臂）；Franka（LIBERO 衍生）与双臂工业 cell。
- **主结果：** GaP 在 pose/geometry 变化列 **0.93–0.99**；π₀.₅ / MolmoAct2 在 permutation/mixed 等列降至 **~0.20**；**π₀.₅ w/ GaP**、**MolmoAct2 w/ GaP** staging 后 **>2×** VLA 裸跑；单 agent 裸 Python CaP 与合并多 agent 消融 **成功率归零**（结构验证/接口不匹配）。
- **工程吞吐：** 周期时间仍低于 ~7 s/instance 工业标准；需减少 VLM 调用、加速 IK/规划。
- **局限：** 准静态 pick-and-place 为主；线缆插入需力传感；可变形体、动力学、移动目标为未来工作。

## 核心论文摘录（MVP）

### 1) 变体自动化（VA）任务类：FA / VA / GR 三分法

- **链接：** <https://arxiv.org/html/2607.05369v1#S2>
- **摘录要点：** **FA** 重复相同运动（点焊、喷涂）；**GR** 开放任务+开放环境（家庭杂务）；**VA** 在 **已知工位、固定机器人与传感器** 下，**持久执行同一任务族** 但 **SKU/几何与位姿有界变化**（分拣包裹、咖啡馆咖啡、商用厨房三明治）。VA 允许复用物体模型、标定传感器与模块化技能，同时仍需适应每个新实例。
- **对 wiki 的映射：**
  - [变体自动化（VA）](../../wiki/concepts/variational-automation.md) — FA/VA/GR 任务谱与假设边界。
  - [GaP 论文实体](../../wiki/entities/paper-gap-graph-as-policy.md) — VA 形式化 tuple $\mathcal{T}=\langle\mathcal{L},\mathcal{E},\mathcal{R},\mathcal{O},\mathcal{X},\mathcal{B},\mathcal{J}\rangle$。

### 2) Graph-as-Policy：计算图即策略

- **链接：** <https://arxiv.org/html/2607.05369v1#S3>
- **摘录要点：** 节点 = 带类型签名的原子原语（取图、感知推理、运动执行、脚本路由）；边 = 数据依赖或控制流（success/failure route）；**skill** = 自然语言规约下由 agent 组装的节点集合。相对单 agent **自由 Python CaP**，图结构限制上下文、支持静态验证、便于 edge 解释器 **无 agent 持久执行**。
- **对 wiki 的映射：**
  - [GaP](../../wiki/entities/paper-gap-graph-as-policy.md) — 与 [ASPIRE](../../wiki/methods/aspire.md)（程序技能库 + trace 调试）、[CaP-X](../../wiki/methods/aspire.md) 的对照轴。
  - [Manipulation 任务](../../wiki/tasks/manipulation.md) — agentic 操作栈新分支。

### 3) 多 agent harness + MORSL 技能库

- **链接：** <https://arxiv.org/html/2607.05369v1#S4>
- **摘录要点：** Orchestrator 分段 → Skill Agents 各写子图 → 聚合布线；MORSL 混合 **model-based**（ROS 过程）与 **model-free**（如 GraspGen）；依赖 Claude/Gemini 等 coding tools。
- **对 wiki 的映射：**
  - [GaP](../../wiki/entities/paper-gap-graph-as-policy.md) — MORSL 与 harness 架构图。
  - [NVIDIA GEAR Lab](../../wiki/entities/nvidia-gear-lab.md) — Jim Fan / Yuke Zhu 署名网络。

### 4) 仿真排练自学习与失败定位

- **链接：** <https://arxiv.org/html/2607.05369v1#S5>
- **摘录要点：** Isaac 仿真渲染感知、物理与接触；并行采样 $\mathcal{B}$ 中位姿/配置；节点前后状态差分定位失败；迭代编辑图直至加权（成功率+吞吐）plateau。
- **对 wiki 的映射：**
  - [GaP](../../wiki/entities/paper-gap-graph-as-policy.md) — Make Popcorn 自学习曲线与三阶段（抓取/搬运/放置）。
  - [Simulation Evaluation Infrastructure](../../wiki/concepts/simulation-evaluation-infrastructure.md) — 仿真作为 agent 内环排练场。

### 5) 8 VA benchmark 与 VLA/TAMP/CaP 对比

- **链接：** <https://arxiv.org/html/2607.05369v1#S6>
- **摘录要点：** 5500+ 试验；GaP **0.95–0.99** vs π₀.₅ **0.15–0.20**（大位姿变化列）；**π₀.₅ w/ GaP** 在 Pack varied **0.67 vs 0.17**；消融：无图单 LLM Python **0%**；多 agent 合并为单 LLM **0%**（结构验证全失败）；真机 grocery fulfillment **25/25** vs TipTop **8/25**。
- **对 wiki 的映射：**
  - [VLA](../../wiki/methods/vla.md) — GaP staging 作为 **可靠性补丁** 而非替代 VLA 泛化。
  - [变体自动化](../../wiki/concepts/variational-automation.md) — benchmark 族定义。
