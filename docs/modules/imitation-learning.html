模仿学习模块

模仿学习的核心不是“把演示喂给网络”，而是把人类或专家策略里的动作先验、技能结构和行为风格，转成机器人能学、能执行、能泛化的控制能力。

这个模块解决什么问题

当任务很难手写 reward、探索空间太大，或者你已经有专家演示数据时，模仿学习能显著降低学习难度，让机器人更快得到可用策略。

为什么对机器人重要

机器人很多高阶技能，本质上带有人类行为先验。无论是遥操作、动捕、专家控制器，还是已有 policy 生成的数据，IL 都是把这些经验注入系统的高效方法。

前置依赖

  • 深度学习基础：监督学习、损失函数、泛化
  • 时序建模：序列、历史窗口、动作条件预测
  • 控制基础：状态反馈、轨迹跟踪、稳定性
  • 动作表示：关节角、root pose、接触、phase、latent skill

你真正要掌握的主线

  • 演示数据从哪来，质量怎么判断
  • Behavior Cloning 为什么简单但容易翻车
  • Distribution Shift 为什么是 IL 的核心痛点
  • DAgger、逆动力学、latent skill、diffusion policy 分别解决什么问题
  • IL 怎么和 RL、tracking control、retargeting 组合起来用

快速导览

先理解模仿学习的基本矛盾：你想复制专家行为，但部署时机器人会偏离专家分布。后面所有方法，基本都在和这件事较劲。

演示数据从哪来

遥操作、动捕、仿真专家、优化器轨迹、历史日志，各有偏差和成本。

核心方法怎么分

BC、DAgger、inverse model、latent skill、diffusion policy，不是一锅粥。

机器人里怎么落地

retarget、tracking、reference motion、policy conditioning，才是真正接地气的部分。

IL 和 RL 怎么协同

现实里强系统经常不是二选一，而是先模仿、再强化、最后部署。

1. 演示数据从哪来，比算法名更重要

垃圾数据进，垃圾策略出。IL 很吃数据分布和动作质量。

遥操作 / Teleoperation

优势是贴近真实任务，能收集操作意图；问题是噪声大、延迟重、动作不一定适合目标机器人直接执行。

动作捕捉 / Motion Capture

适合 humanoid 技能和自然动作风格迁移，但人和机器人形态不一致，必须经过 retargeting、动力学约束和接触修正。

仿真专家 / 规划器 / MPC

优点是可批量生成、标签稳定；缺点是继承了上游控制器的偏置，而且未必覆盖复杂恢复行为。

已有策略回放数据

你可以用已有 RL policy 或专家 policy 生成 rollouts，再训练更轻、更稳、更快部署的 student policy。

2. 核心方法不是一个桶

它们解决的问题不同。别把所有“从演示学”的方法都叫 BC。

Behavior Cloning

最直接 · 最常见 · 也是最容易分布偏移

把专家状态映射到专家动作，按监督学习做。优点是简单、稳定、快；缺点是部署时一旦偏离专家轨迹，误差会连锁放大。

DAgger

解决 distribution shift 的经典方法

让 learner 自己跑，再让专家给纠正标签，把 learner 真会到达的状态也纳入训练。代价是需要在线专家反馈。

Inverse / Forward Model

建模动作-结果关系

常用于表示学习、动作恢复和控制辅助。特别适合处理观测和动作之间的时序因果结构。

Diffusion / Sequence Policy

适合复杂多峰行为

当一个状态下存在多种合理动作时，扩散或序列生成模型往往比单点回归更自然，尤其适合 manipulation 和长时序技能。

3. 机器人里的 IL，重点是动作表示和落地管线

论文里一句 “train policy from demonstrations”，落地时背后通常是一整条很长的工程链。

Retargeting

人体动作不能直接塞给机器人。需要处理骨架比例、自由度差异、接触约束、关节极限和动力学可执行性。

Reference Motion Tracking

很多 humanoid 技能学习，不是纯 BC，而是让 policy 在奖励里跟踪参考动作。这其实是 imitation 和 RL 的混合范式。

Conditioned Policy

给策略输入 phase、future reference、goal token、skill embedding，可以把“会做动作”提升成“可控地做指定动作”。

动作平滑与可部署性

演示数据看起来很帅，不代表机器人真能跟。最终部署时要考虑带宽、关节冲击、稳定裕度、延迟和安全约束。

4. IL 和 RL 不是对立关系

很多最好用的系统，都是先靠 IL 拿到行为先验，再用 RL 做鲁棒性和性能提升。

先 BC，再 RL Fine-tune

先学会“像样地动”，再让 RL 优化速度、能耗、抗扰和任务完成度。这是非常常见的路线。

Tracking Reward + RL

让策略跟踪参考动作，同时保留 RL 的探索能力。对 humanoid 动作风格学习尤其有效。

Teacher-Student Distillation

让复杂专家（MPC、大模型策略、多模态控制器）教一个轻量 student policy，适合部署侧压缩。

5. 人形机器人场景下，IL 特别有价值的地方

因为很多“自然动作”很难直接靠 reward 搜出来，但人类演示能提供极强先验。

Locomotion Style

不仅要走，还想走得像人、协调、自然，这时参考动作和 imitation signal 很关键。

Recovery / Agile Skills

摔倒恢复、快速转身、侧移、跨步等技能，单靠 reward 搜索很痛苦，演示能大幅降低难度。

Manipulation with Intent

抓、放、插、开关门这类带语义结构的任务，演示数据往往比纯探索更高效。

Multi-skill Composition

多个 skill 之间切换和组合，可以借助 latent embedding、skill token 或 reference library 来做。

7. 常见坑点

模仿学习看着比 RL 简单，实际很容易被数据和分布问题狠狠干碎。

以为数据多就够

没覆盖关键恢复状态，再多数据也可能一偏就崩。

忽视演示质量差异

专家数据如果混了大量迟滞、犹豫、误操作，模型会老实学进去。

只追求动作拟合误差低

离线 loss 很低，不代表闭环执行效果好。机器人里闭环稳定性永远比单步回归漂亮更重要。

retarget 不认真做

人类动作很优雅，但机器人身体不是人。retarget 做烂了，后面学什么都歪。
