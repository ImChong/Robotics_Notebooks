## 摘录 1：问题与主张（§I / Abstract）

- **痛点：** 基准上好看的人形 VLA，在零售等真实场景仍卡在执行误差、分布偏移与环境变化；naive fine-tune 常直接失败。
- **主张：** lab→store 鸿沟主要是 **系统集成 / 数据设计**，不是再发明架构。DEED = **(1) Data-Efficient 后训练配方** + **(2) Experience-Driven（RECAP 适配）** + **(3) latent in-/OOD 诊断工具**。
- **任务：** 超市薯片补货——从箱中抓袋、转身放到货架，再转回；左臂为主，Unitree 自带 locomotion 保平衡。

## 摘录 2：两阶段方法（§II）

### Data-Efficient（DE-）

- **频率层级：** \(f_r = f_{\mathrm{ctrl}} \le f_{\mathrm{cam}}\)，\(f_t \ge f_r\)；本文 \(f_t=50\)、\(f_{\mathrm{cam}}=30\)、\(f_r=f_{\mathrm{ctrl}}=25\) Hz。
- **数据策展：** 状态覆盖均衡、只留高效成功、保留附带恢复、动作一致性、连续多子任务 episode（不切碎过渡）等规则。
- **视觉：** 适配 **IA-VLA** 用 VLM 掩膜/框高亮任务相关区域（货架空位等），减轻互联网预训练骨干的虚假相关。
- **降依赖技巧：** 手部用 **二进制开合** 而非连续灵巧关节；推理期 **Butterworth** 平滑 action chunk；语言提示避免子任务共用弱措辞。
- **动作空间：** 原生 32 维 → **20** 维（每臂 7 关节 + 每手二进制 + 1 腰 + 3 基座速度）；头 + 双腕 RealSense RGB。

### Experience-Driven（-ED）

- 自 **RECAP**（π\*0.6 系）适配到 GR00T **解耦**（VLM 骨干 + Flow-Matching 动作头）：用文本前缀 `Advantage=True/False` 条件化，价值函数只看视觉+语言（Eagle-3 + MLP，101-bin 离散回报）。
- 每轮 refinement **从原公开 ckpt 重启**（非接着上轮策略），混入 teleop + 自主成功/失败 + 人类纠正；推理固定 `Advantage=True`。

**对 wiki 的映射：** 实体页画 DE → RECAP 闭环 flowchart；强调「不改模型结构」。

## 摘录 3：实验要点（§III）

| 设定 | 数字 |
|------|------|
| 平台 | Unitree G1-Edu + Dex-3；单卡 **RTX 5090** |
| 初始化 | `GR00T-N1.6-G1-PnPAppleToPlate` |
| Teleop | **81** 条 ≈ **51.5 min**（单品类薯片） |
| 自主 rollout（两轮 RECAP） | **116** 条（41 成功 / 75 失败）≈ 56.9 min；合计操作 ≈ **108.4 min** |
| Naive SFT | **0%**（0/50） |
| DE policy | **32%**（16/50），均值 24.30 s/袋，最多连续 4 袋 |
| RECAP iter 1 | **42%**（21/50），22.37 s；最多连续袋数降为 1 |
| RECAP iter 2 | **22%**（11/50）— 自生成数据主导 → 分布漂移 |
| 局限 | 单任务/单平台、二值成功、评测 episode 少、reset 行为被 advantage 抑制需外部复位 |

**对 wiki 的映射：** 用「数据质量 > 数据量；经验驱动需保持 teleop 混合比」写清工程警告。
