## 3. 首页区块草稿

### 3.1 这个项目是做什么的？
这个项目不是机器人资料堆砌，也不是单纯的思维导图展示。
它更像一个面向长期成长的 **技术导航系统**：

- 帮你看清机器人领域的核心模块
- 帮你理解各模块之间的依赖关系
- 帮你知道当前该学什么、下一步该补什么
- 帮你从一个切入口，逐步走向全栈能力

### 3.2 适合谁？
- 想系统建立机器人知识框架的人
- 想从运动控制切入机器人算法的人
- 想进入人形机器人、强化学习、模仿学习方向的人
- 想从“会看论文”走向“能做系统”的人
- 想最终成长为全栈机器人工程师的人

### 3.3 当前重点方向
- 运动控制
- 强化学习
- 模仿学习
- 全栈扩展

### 3.4 两条成长路线
- 主路线：运动控制算法工程师成长路线

### 3.5 项目结构
- `wiki/`：结构化知识页
- `roadmap/`：成长路线与学习阶段
- `tech-map/`：技术栈地图与模块依赖
- `references/`：论文导航与开源生态
- `sources/`：原始资料索引

### 3.6 项目边界
- `Humanoid_Robot_Learning_Paper_Notebooks`：单篇论文深读，负责“点”
- `Robotics_Notebooks`：技术栈、路线、模块组织，负责“线和面”
- `ImChong.github.io`：对外展示，负责“展示层”

## 4. 风格对齐建议

为了保持与另外两个项目一致，首页建议遵循以下风格：

### 视觉风格
- 白底 / 深色模式切换
- 中等宽度内容容器（860px~960px）
- 顶部 sticky header
- 卡片式模块区块
- 字体风格保持简洁学术风

### 信息风格
- 不写大段空话
- 每个区块都要可执行、可导航
- 首页突出“项目定位 + 路线 + 地图 + 入口”
- 延续简历站的清爽卡片感 + 论文站的结构化导航感

### 技术风格
- 优先纯 HTML + CSS + JS
- 保留双语扩展能力
- 兼容 GitHub Pages
