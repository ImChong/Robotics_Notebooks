运动控制模块

这是整个 `Robotics_Notebooks` 当前最核心的支点页面。你现在的切入口是运动控制算法工程师，所以这页不只是一个概念介绍页，而应该成为整个网站的硬骨架。

这个模块解决什么问题

本质问题

让机器人在给定任务、模型、约束和扰动条件下，产生稳定、可控、可执行的身体运动行为。

在人形机器人里的具体含义

不是简单地“让关节动起来”，而是要在接触切换、浮动基、平衡约束、关节限制、驱动能力和外界扰动下，持续生成可落地的控制输出。

为什么它是硬骨架

因为无论后面是强化学习、模仿学习，还是感知、规划、部署，最终都要落到控制执行层。没有运动控制，后面的智能层没有真正的落点。

为什么对你当前路线最重要

它是当前切入口

你的职业切入口就是机器人运动控制算法工程师，所以这块不是可选项，是主战场。

它决定后续上限

控制理解得浅，后面学 RL / IL 很容易变成只会调参，不懂系统本质。

它连接传统方法与学习方法

传统控制、优化控制、WBC、MPC 和 RL / IL 不是互斥关系，而是连续谱。

它最容易形成工程闭环

相比泛泛学视觉或规划，控制更容易从模型、仿真、实验、部署一路闭环起来。

前置依赖图

第一层：数学基础

线性代数、微积分、概率统计、优化基础。

第二层：机器人学基础

刚体运动、坐标系变换、运动学、动力学。

第三层：状态与约束

状态估计、接触建模、约束理解、执行器能力边界。

第四层：控制方法

PID / LQR → OCP / MPC / QP → WBC / TSID → Humanoid Locomotion。

核心子主题

状态估计

解决“机器人现在到底处于什么状态”的问题，是一切控制的入口。

经典控制

PID、LQR 等方法帮助建立最基础的控制直觉和闭环认知。

最优控制 / MPC

把模型、约束、目标函数和未来预测引入控制问题。

QP-WBC / TSID

处理多任务、多约束和全身控制问题，是人形控制的关键中枢。

质心动力学 / 步态控制

直接关联双足平衡、步态生成和扰动恢复。

控制与学习结合

理解传统控制如何与 RL / IL 协同，而不是把两者对立起来。

学完后应该会什么

懂系统结构

知道一个人形控制系统从状态估计、模型、任务约束到控制输出怎么串起来。

懂方法边界

知道 PID、LQR、MPC、WBC 各自擅长什么、局限什么，不会把它们混成一团。

懂论文结构

能读懂 humanoid locomotion / WBC / MPC 论文中的控制框架，不再只看结论图。

懂如何往 RL / IL 过渡

知道学习方法不是替代控制，而是在某些层面增强控制能力。

最小落地任务

任务 1：模型理解

用 Pinocchio 或类似工具导入一个人形模型，跑通正逆运动学与基本动力学量读取。

任务 2：控制直觉

实现一个倒立摆 / 二阶系统的 PID、LQR 或 MPC 控制，对比不同方法的表现。

任务 3：论文拆图

读一篇 humanoid locomotion / WBC 论文，把其控制系统结构重新画成框图。

任务 4：仿真闭环

在仿真环境里跑通一个最基础的 locomotion 或 balance baseline，并记录问题来源。

和 RL / IL 的关系

和强化学习的关系

RL 不是对控制的替代，而是让你在高维、复杂、难建模任务中学习策略。没有控制骨架，RL 很容易流于黑箱调参。

和模仿学习的关系

IL 让技能迁移更快，但最终也要落到控制可执行性上。动作数据不是魔法，还是得落在动力学和执行边界里。

更合理的理解

控制是骨架，RL / IL 是能力增强层。先懂骨架，再扩智能，路线才稳。

下一步学习建议

如果你现在基础还不稳

先补线性代数、刚体运动、运动学和动力学，别急着直接冲 RL 论文。

如果你已经在做控制

重点把状态估计、MPC、WBC 和 humanoid locomotion 的结构理解打深。

如果你准备往学习方向扩

在控制主线稳定之后，再把 RL / IL 接进来，这样会比一上来就端到端更稳。
