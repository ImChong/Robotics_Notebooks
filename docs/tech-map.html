技术栈地图

当前主干不是“覆盖全”，而是“依赖关系清楚、重点主线明确、扩展边界清晰”。

核心依赖主链

数学 → 机器人学 → 控制

线性代数 → 刚体运动 → 运动学 → 动力学 → 全身控制。

概率 → 状态估计

概率统计 → 滤波 → 状态估计 → 部分可观测建模。

优化 → 最优控制

优化基础 → OCP → MPC / QP-WBC。

深度学习 → 学习控制

深度学习 → 强化学习 / 模仿学习 → humanoid skill learning。

当前主攻与未来扩展

当前主攻：运动控制

运动控制、强化学习、模仿学习、人形机器人。

未来扩展

感知、规划、ROS2 / Middleware、部署、硬件系统、系统集成。

技术地图职责

不是替代知识页，而是为知识页提供结构和依赖导航。
