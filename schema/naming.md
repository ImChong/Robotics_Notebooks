# Naming Rules

## 总原则

命名追求：
- 清晰
- 可搜索
- 稳定
- 尽量避免中英文混杂在文件名中

## 目录命名

统一使用小写英文 + 连字符：

- `overview/`
- `concepts/`
- `methods/`
- `tasks/`
- `comparisons/`
- `roadmaps/`

## 文件命名

统一使用小写英文 + 连字符 + `.md`：

示例：
- `robot-learning-overview.md`
- `sim2real.md`
- `whole-body-control.md`
- `reinforcement-learning.md`
- `wbc-vs-rl.md`

## 页面标题

文件名用英文，页面内容里可以同时保留中英文标题。

推荐格式：
```md
# Sim2Real
**仿真到现实迁移**
```

## 避免事项

不要：
- 用空格做文件名
- 用时间戳做知识页文件名
- 把多个不相关主题塞进同一个文件
- 把 README 继续当总索引和总内容的混合垃圾场
