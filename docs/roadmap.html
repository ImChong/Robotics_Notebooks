成长路线

这页不是给你一份看起来很全的课程目录，而是把“从当前能力出发，怎样一步步长成更强的人形机器人控制与学习工程师”讲清楚。

1. 这条路线为谁准备

当前画像

适合已经有一定机器人基础，但想把运动控制、强化学习、模仿学习和人形机器人主线真正打深的人。

目标画像

不是泛泛懂一点，而是能在控制、学习、仿真和部署之间形成稳定闭环，逐步成长为更强的人形机器人算法工程师。

路线原则

先打主线，再补扩展；先做出闭环，再扩知识面；先形成判断力，再追求资料覆盖率。

2. 路线 A：控制与学习主线

这是当前最核心、最值得投入的主线，也是你项目内容建设的主骨架。

L0 数学基础

线性代数、微积分、概率统计、优化基础。目标不是把数学做成孤岛，而是让它能服务控制、估计和学习。

L1 机器人学骨架

刚体运动、坐标变换、运动学、动力学。目标是能读懂机器人系统怎么描述，而不是只会抄公式。

L2 控制基础

PD / PID、LQR、MPC、QP 和控制问题建模。目标是建立闭环、稳定性、约束和系统结构直觉。

L3 人形运动控制

质心动力学、步态控制、全身控制、接触切换、平衡与扰动恢复。目标是把 humanoid locomotion 主线真正打透。

L4 机器人强化学习

从 MDP 建模、PPO / SAC 到 reward design、curriculum、domain randomization 和 sim2real，目标是建立机器人 RL 的完整工程视角。

L5 机器人模仿学习

从 behavior cloning、distribution shift 到 motion retargeting、reference tracking、diffusion policy，目标是建立技能学习和行为先验注入能力。

3. 路线 B：系统能力扩展主线

在主线稳定后，再逐步向系统化能力扩展，不要一开始就把自己摊太平。

感知与视觉

视觉、点云、多模态、状态感知和任务感知，会在后续真实系统集成里越来越重要。

规划与决策

从低层控制走向任务层结构，理解 motion planning、task planning 和层级系统如何衔接。

00 - Robotic Manipulation

ROS2 / Middleware / 通信

这是把算法塞进真实机器人系统的基础设施，不补这层，很多能力会长期停在单机实验里。

硬件、部署与系统集成

驱动、执行器、安全、调试、工程闭环，决定你能不能把算法真正落到整机。

4. 每个阶段应该达到什么状态

早期目标

不再把控制、RL、IL 看成互相割裂的三个词，而是知道它们在系统里各自站什么位置。

中期目标

能复现一个最小 humanoid / locomotion 训练闭环，能读懂主流论文的系统结构，不只是看结果图。

进阶目标

能对 observation、action、reward、control interface、sim2real 路径给出自己的判断，而不是照抄已有配置。

长期目标

形成从理论、控制、学习到工程部署的连续理解，逐步向更完整的人形机器人系统能力扩展。

5. 现在最值得优先做什么

优先动作 1

持续深化三大模块页，把运动控制、强化学习、模仿学习写成真正能反复回看的总览页。

优先动作 2

围绕 locomotion 主线补关键关系页，例如 WBC vs RL、RL vs IL、Sim2Real pipeline、retargeting pipeline。

优先动作 3

从 resources 里挑最有价值的理论笔记、训练笔记和 know-how，逐步挂回主线页面形成内容闭环。

优先动作 4

每补一页，都尽量回答“我为什么要学它”和“它在人形机器人系统里处于哪一层”。

6. 最常见的跑偏方式

跑偏 1

知识面摊太开，结果控制主线不深，什么都知道一点，但没形成真正能落地的判断力。

跑偏 2

过早沉迷网页形式或资料收集，而没有持续产出结构化内容页。项目会越来越像展示页，而不是知识系统。

跑偏 3

跳过控制骨架，直接端到端冲 RL / IL，最后容易停在“调参能跑”但无法解释系统行为的层面。

7. 一句话总结

成长路线的重点，不是把主题越列越多，而是让你沿着控制 → 学习 → 部署这条主线，持续长出真正可用的系统能力。
