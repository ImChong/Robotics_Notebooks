# legs_arxiv_2606_01458

> 来源归档（ingest）

- **标题：** LEGS: Fine-Tuning Teleop-Free VLAs for Humanoid Loco-manipulation in an Embodied Gaussian Splatting World
- **类型：** paper
- **来源：** arXiv:2606.01458（2026-05-31）
- **作者：** Hojune Kim, Timothy Chen, Jiankai Sun, Lars W. Osterberg, Qianzhong Chen, Ke Wang, Mac Schwager（Stanford University）
- **入库日期：** 2026-06-04
- **一句话说明：** 混合仿真器 **LEGS**：SAM3D/URDF **mesh 前景** + 手持视频 **3DGS 背景** + 两阶段颜色校准；MuJoCo + **SONIC** 产生 **无遥操作** 程序化 loco-manip 演示；motion 与 appearance 解耦可 **GPU 重渲染** 增广；在 Unitree G1 上微调 ψ0 / π0.5 / GR00T N1.6，**纯 LEGS 数据** 匹配或超过 50-demo teleop，并显著优于 mesh-only 合成基线。

## 核心论文摘录（MVP）

### 1) 问题与总贡献（Abstract / §1）

- **链接：** <https://arxiv.org/abs/2606.01458>
- **核心贡献：** 针对人形 **loco-manipulation VLA 微调** 的数据成本：提出 **LEGS**（Loco-manipulation via Embodied Gaussian Splatting）——前景 mesh（机器人、物体、道具）叠在由手持场景视频重建的 **3DGS 背景** 上；**程序化 motion primitive** 大规模合成带标签演示；**确定性两阶段颜色校准** 对齐部署相机。在 G1 上三个递难 pick-place 与三个 VLA backbone 上，**仅 LEGS 数据** 的训练策略 **匹配或超过** 50 条 human teleop；相对 **SAM3D mesh-only**（去掉 3DGS 与校准）在全部实验上更优；**LEGS-aug** 在物体+场景外观联合偏移下保持成功而 teleop 完全失败。

### 2) 视觉前端与颜色校准（§3.1）

- **链接：** <https://arxiv.org/html/2606.01458>
- **核心贡献：** 背景：~1–2 分钟手持视频 → COLMAP + 3DGS；前景：URDF + SAM3D 物体/道具 mesh；每帧深度合成。校准：关闭 AE/AWB；**per-mesh 线性 RGB 缩放 R** 修正 SAM3D 反照率 + **全局 3×3 矩阵 M**（ColorChecker 最小二乘）把合成图映射到 **RealSense D435 部署相机**。
- **对 wiki 的映射：**
  - [GS-Playground](../../wiki/entities/gs-playground.md)（同为 3DGS 视觉仿真脉络）
  - [Sim2Real](../../wiki/concepts/sim2real.md)

### 3) 物理后端、SONIC 接口与程序化 episode（§3.2–3.3）

- **核心贡献：** MuJoCo 500 Hz 与渲染解耦；**SONIC** 作为低层 WBC（仿真与真机同一二进制）；高层 **18-D 命令**（每臂 6-D 腕 SE(3) + 夹爪标量 + 底座 $v_x,v_y,\omega_z,h$）。任务分解为 **Walk / Pick / Place** 等 parametrized primitive，仅保存验证成功 episode；记录的命令流与渲染输入无关 → 可 **仅 GPU 重渲染** 换背景/物体/提示词。
- **对 wiki 的映射：**

  - [Whole-Body Control](../../wiki/concepts/whole-body-control.md)
  - [Teleoperation](../../wiki/tasks/teleoperation.md)（对比 teleop 数据成本 Table 1）

### 4) 实验：三任务 × 三 VLA × 数据条件（§4）

- **任务：** Task 1 静止桌面；Task 2 走到桌边再 pick-place；Task 3 走–拿–转身–到低桌蹲放（长时程 loco-manip）。
- **数据：** Teleop(50)；SAM3D(200) mesh-only 消融；LEGS(200) 主设置；LEGS(50) 预算匹配 teleop。
- **骨干：** ψ0、π0.5、GR00T N1.6（均用 SONIC + 18-D 接口微调）。
- **关键数字（end TSR /10）：** Task 3 上 Teleop(50) 三 backbone **均为 0**；LEGS(200) 为 **5、2、6**；SAM3D(200) 平均约 **33%** vs LEGS(200) **67%**；外观随机化下 **LEGS-aug** 在最难 **objects+scene** 上优于 **SAM3D-aug**，teleop 无法重渲染参与增广。
- **对 wiki 的映射：**
  - [π0.7 Policy](../../wiki/methods/pi07-policy.md)、[Unitree G1](../../wiki/entities/unitree-g1.md)
  - [VIRAL](../../wiki/entities/paper-viral-humanoid-visual-sim2real.md)（同属 loco-manip 视觉 sim 数据路线，方法不同）

### 5) 局限（§5）

- 颜色映射假设 **固定 AE/AWB**，对扫描外光照敏感；新场景仍需手持视频；3DGS 在 **高反光/透明/动态背景** 上受限；程序化 motion 非动态最优；外观随机化仅 **小物体** 尺度；单平台 **G1 + D435 + SONIC**。
