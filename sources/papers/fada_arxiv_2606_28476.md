# FADA: Few-Shot Domain Adaptation via Dynamics Alignment for Humanoid Control（arXiv:2606.28476）

> 来源归档（ingest）

- **标题：** FADA: Few-Shot Domain Adaptation via Dynamics Alignment for Humanoid Control
- **缩写：** **FADA**
- **类型：** paper / humanoid control + few-shot domain adaptation + sim2real
- **arXiv：** <https://arxiv.org/abs/2606.28476>
- **PDF：** <https://arxiv.org/pdf/2606.28476>
- **项目页：** <https://lecar-lab.github.io/FADA-humanoid/>
- **发表日期：** 2026-06-26
- **作者：** Angchen Xie, Nikhil Sobanbabu, Ishayu Shikhare, Alan Wang, Max Simchowitz, Guanya Shi
- **机构：** 卡内基梅隆大学（CMU）
- **入库日期：** 2026-07-01
- **一句话说明：** 三阶段 **Planner–IDM** 框架：仿真中特权 oracle → DAgger 蒸馏为 **规划器（短视界本体感知意图）+ 逆动力学模型 IDM（意图→动作）**；部署时 **冻结规划器**，仅用约 **2 分钟** 目标域 rollout 的 **(观测, 动作) 对** 监督微调 **IDM（LoRA）**，无需目标奖励、专家演示或仿真重标定，即可对齐目标域动力学。

## 核心论文摘录（MVP）

### 1) 问题：动力学失配改变「如何执行」而非「做什么」（Abstract / §1）

- **链接：** <https://arxiv.org/abs/2606.28476>
- **核心贡献：** 高精度人形控制在地形、载荷、执行器响应变化时，**同一控制意图** 可能产生不同身体运动。现有路线要么 **零样本鲁棒**（域随机化、历史条件 in-context 适应）但不针对目标域特化，要么 **重适应管线**（SysID、残差动力学、全策略微调）需要奖励、专家或仿真拟合。FADA 问：**能否不用目标奖励或精确先验模型，用极少目标域数据做 few-shot 适应？**
- **对 wiki 的映射：**

  - [Sim2Real](../../wiki/concepts/sim2real.md)

### 2) Planner–IDM 分解与源域训练（§3–4.1）

- **链接：** <https://arxiv.org/html/2606.28476>
- **核心贡献：**
  - **Planner $P$**：由命令 + 历史观测预测 $K$ 步未来本体感知 $\hat{Y}_t^K$（默认 $K=6$，$H=30$）。
  - **IDM $I$**：由历史 + 预测未来映射到动作 chunk $\hat{U}_t^K$；**receding-horizon** 只执行首动作。
  - **IDM 监督**：用 rollout 上 **实际执行** 的 $(Y_{exec}^K, U_{exec}^K)$ 做逆动力学 MSE（首动作），**不必是 oracle 最优动作**。
  - **Planner 监督**：经 **stop-gradient IDM** 优化，使 planner 预测的未来经 IDM 后 **动作与 oracle 首动作一致**（action-grounded futures，非直接回归 oracle 观测轨迹）。
  - **Oracle relabeling**：学生 rollout 访问状态快照，事后在仿真中 roll oracle 得 $(Y_{orac}^K, U_{orac}^K)$ 供 planner 训练。
  - **架构**：双 Transformer；源训练在 **IsaacSim** + 域随机化。
- **对 wiki 的映射：**
  - [DAgger](../../wiki/methods/dagger.md)
  - [Domain Randomization](../../wiki/concepts/domain-randomization.md)

### 3) Few-shot 目标域适应：只微调 IDM（§4.2）

- **核心贡献：**
  - 用源策略 $(P, I)$ 在目标域收集 rollout，提取窗口 $\mathcal{W}_{tgt}$。
  - **冻结 planner 与 IDM 预训练权重**，仅优化 **IDM 上 LoRA 适配器** $\Delta\psi$（Eq. 4.5 同 IDM 首动作 MSE）。
  - **数据预算**：locomotion 约 **2 min @ 50 Hz（≈6000 步）**；全身跟踪约 **6 次 ≈20 s 参考动作重复**。
  - **无需**：目标奖励、特权标签、仿真重标定、在线 RL、MPC。
- **对 wiki 的映射：**
  - [Sim2Real 方法对比](../../wiki/comparisons/sim2real-approaches.md)
  - [RMA](../../wiki/entities/paper-rma-rapid-motor-adaptation.md)（对照：in-context vs 权重更新）

### 4) 实验：G1 + T1 真机与 IsaacSim→MuJoCo（§5）

- **平台：** Unitree G1（29 DoF）、Booster T1（23 DoF）。
- **真机任务（Table 1，5 次试验平均）：**
  - G1 斜坡直线行走（双 15° 坡）：成功率 **20%→80%**（FADA-zs→FADA）；TF-DAgger **0%**。
  - G1 非对称 ~3 kg 载荷 loco：$\bar{E}_v$ **1.289→0.797**（归一化）。
  - G1 软垫功夫：$\bar{E}_{mpjpe}$ **1.18→0.86**。
  - T1 非对称 1 kg 臂载 loco：$\bar{E}_v$ **1.043→0.866**。
  - T1 拉 ~6 kg 洗衣篮：成功率 **20%→100%**。
  - **平均**：成功率任务 **20%→90%**；跟踪误差任务相对 FADA-zs **降 27.4%**。
- **Sim-to-sim（IsaacSim→MuJoCo，5 任务）：** FADA 平均归一化误差较 FADA-zs **降 24.7%**、较 TF-DAgger **降 26.8%**；TF-CoPred-ft（只微调未来观测预测）反而变差。
- **消融：** $K=6$ 优于 $K=1$（约 **18%** 误差降）；**LoRA** 优于全参 IDM 微调（易过拟合）；6000 步达性能平台。
- **控制实验（固定基座 7-DoF 臂 + 腕部载荷 0–5 kg）：** 冻结 planner 后 IDM 适应使跟踪误差 **≈24%↓**、consistency gap **≈54%↓**；planner 对载荷几乎不变（**≈7%** RMSE 变化）→ 验证 **意图 vs 执行** 分离。
- **对 wiki 的映射：**
  - [Unitree G1](../../wiki/entities/unitree-g1.md)
  - [Whole-Body Control](../../wiki/concepts/whole-body-control.md)

### 5) 局限（§5 / Conclusion）

- 未显式建模全身 **人–物交互**（loco-manipulation 接触丰富任务）。
- 适应仅用本体感知轨迹，**无视觉/触觉**。
- 需要零样本策略 **足够可部署** 以收集目标 rollout（依赖源域 DAgger 质量）。
